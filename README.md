# HTML 转 Figma 助手

一个 Chrome 扩展，用于采集当前网页或本地 HTML 页面，并导出为可用于 Figma 设计稿转换的 JSON 文件。

## 功能

- 一键采集当前标签页页面结构和样式。
- 支持网页和本地 HTML 页面。
- 支持跨域图片代理模式，降低图片丢失概率。
- 支持设置图片采集并发数，便于在速度和稳定性之间取舍。
- 采集时会自动忽略插件自身的浮层卡片。

## 安装

### 从源码加载

1. 下载或克隆本仓库。
2. 打开 Chrome，进入 `chrome://extensions/`。
3. 打开右上角的「开发者模式」。
4. 点击「加载已解压的扩展程序」。
5. 选择本项目目录：`html-to-figma-capture-extension`。
6. 安装完成后，浏览器右上角会出现扩展图标。

### 使用已打包文件

如果你本地已有 `html-to-figma-capture-extension.crx`，可以在 `chrome://extensions/` 页面开启「开发者模式」后拖入安装。

## 使用方法

1. 打开需要采集的网页或本地 HTML 页面。
2. 点击浏览器右上角的扩展图标。
3. 页面右上角会出现「HTML 转 Figma 助手」工具卡片。
4. 按需调整「跨域图片代理模式」和「图片采集并发」。
5. 点击「开始采集」。
6. 采集完成后，浏览器会下载一个 `figma-capture-时间戳.json` 文件。
7. 将下载的 JSON 文件用于后续 Figma 导入或转换流程。

## 配置说明

### 跨域图片代理模式

默认关闭。开启后，插件会通过后台脚本拉取页面图片，可以减少跨域图片采集失败或丢图的情况。

开启后的代价是采集速度可能变慢，尤其是页面图片数量较多时。

### 图片采集并发

只在开启「跨域图片代理模式」后生效。

可选值包括：`4`、`6`、`8`、`10`、`12`、`16`、`20`、`无限`。

建议：

- 图片多、网络不稳定时使用 `4` 或 `6`。
- 一般页面使用默认值 `8`。
- 网络稳定、图片较少时可以提高到 `12` 或以上。

## 常见问题

### 采集结果里出现了插件工具卡片

当前版本已处理这个问题。采集脚本会自动忽略带有 `data-figma-capture-ignore` 标记的插件浮层。

如果仍然遇到，请确认浏览器扩展页面中加载的是最新版本，并重新点击「重新加载」。

### 图片丢失或显示异常

可以开启「跨域图片代理模式」后重新采集，并适当降低「图片采集并发」。

### 点击开始采集后没有下载文件

请检查：

- 当前页面是否已加载完成。
- Chrome 是否拦截了下载。
- 扩展是否在 `chrome://extensions/` 中正常启用。
- 当前网页是否允许扩展注入脚本。
